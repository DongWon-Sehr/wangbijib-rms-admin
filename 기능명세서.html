# 왕비집 예약관리 시스템 v2.0 기능명세서 (Final Revised 5) ## 1. 개요 (Overview) * **프로젝트명**: 왕비집 예약관리 시스템 고도화 (v1.4) * **목적**: 왕비집 전 지점의 예약 관리, 슬롯 최적화, 예약금 관리 및 고객 응대 자동화 효율성 증대. * **플랫폼**: Google Workspace 기반 웹 애플리케이션 (Google Sheets + Google Apps Script). * **배포 설정**: * 실행 권한: `Me` (소유자 권한으로 스크립트 실행) * 접근 권한: `Every Google Account` (모든 구글 계정 사용자 접근 가능 - 내부 인증 로직으로 통제) * **핵심 가치**: "예약 슬롯의 유기적 확보" (예약 확정 시 위젯 캘린더 슬롯을 자동으로 확보하여 매출 증대 기여). ## 2. 시스템 아키텍처 & 기술 스택 ### 2.1. Frontend (SPA) * **Framework**: `Vue.js 3` (Composition API, CDN 방식). * **CSS Framework**: `Tailwind CSS` (CDN 방식). * **Icons**: `Phosphor Icons`. * **Rich Text Editor**: `Quill.js` (메일 템플릿 편집용). * **Notification**: KaKao JS SDK (알림톡 전송). * **구조 전략**: * **Template 분리**: `index.html` 유지보수를 위해 `LayoutHeader.html`, `ViewList.html`, `ModalReservation.html` 등 컴포넌트 단위로 파일 분리 후 `include()` 처리. * **데이터 캐싱 & 동기화**: * **Initial Load**: 최초 진입 시 스피너를 띄우고 전체 예약 데이터 및 설정(Branch Map, Slot Map, Mail Templates)을 메모리에 로드. * **Reactive Update**: 데이터 수정 시 로컬 상태(State)를 즉시 업데이트하여 UI 반응성 확보 후 백그라운드 서버 동기화. * **Background Sync**: 1분 주기로 서버 데이터를 조용히 페치(Fetch)하여 변경 사항을 로컬 상태에 병합. ### 2.2. Backend (Google Apps Script) * **패턴**: `Singleton Pattern` 적용. `Config.gs`, `Util.gs`를 포함한 모든 서비스 파일은 `const` Service = {} 객체 리터럴 구조로 변경하여 인스턴스 관리 비용 제거. (단, `ReservationBatch.gs`, `WebApi.gs`는 함수형 유지) * **API Wrapper 패턴**: `WebApi.gs` 내 `_executeApi(apiName, action, params)` 래퍼 함수 도입. * 모든 API 요청의 진입/성공/실패/에러를 통일된 포맷으로 로깅. * **Error Handling**: Service 계층에서 에러 발생 시 래퍼가 포착하여 표준 실패 응답 반환. * **데이터 액세스**: **Header Name 기반 Object Mapping** (ORM 유사 방식) 적용. `Util.getSheetDataAsObjects(sheetName)`. * **인증(Auth)**: **Google Session 인증** (`Session.getActiveUser().getEmail()` 사용). 기존 비밀번호 방식 폐기. * **템플릿 엔진**: **구글 시트(`mail_template`)**를 DB로 사용하여 HTML 템플릿을 관리. WebApi를 통해 조회 및 업데이트. ### 2.3. 디자인 시스템 (Color Palette) * **Main Color (Royal Red)**: `#8B0000` - 왕비집의 정체성. * **Sub Color (Cool Gray)**: `#1F2937` - 텍스트 및 네비게이션. * **Accent Color (Amber)**: `#D97706` - 중요 알림, 뱃지. * **Background**: `#F3F4F6` - 대시보드 배경. ## 3. 통신 프로토콜 (API Interface) ### 3.1. 표준 응답 포맷 (Standard Response) `Util.createResponse()`를 통해 아래 JSON 구조 준수. ``json { "success": boolean, "data": any, "message": string } `` ### 3.2. Util 메서드 정의 * **`Util.createResponse(success, data, message)`**: 표준 응답 객체 생성. * **`Util.getSheetDataAsObjects(sheetName)`**: 시트 데이터를 JSON 객체 배열로 반환. ## 4. 데이터베이스 스키마 (Google Sheets) 모든 테이블은 데이터 무결성 및 프로필 아이콘 및 팝업 (이름, 이메일, 권한 표시). ### 5.2. 예약 리스트 (Tab: List) * **검색 & 필터**: * 기간(Datepicker), 지점(Badge), 상태(예약/취소/전체), **예약금 상태(Dropdown)**. * **통합 검색창**: 지점명, 이름, 이메일, 메모(고객/관리자) 실시간 필터링. * **UI 구성**: * `고객` 메모 / `관리자 메모` 컬럼 분리. * 헤더 클릭 시 정렬 토글 (예약금 상태 포함). * **모바일 뷰**: * 카드 리스트 **상단**에 컨트롤 바(정렬, 필터) 배치. ### 5.3. 예약 상세/수정 (Modal) * **모드**: `View Mode` / `Edit Mode`. * **읽음 처리**: 모달 오픈 시 또는 **카카오톡 전송 시**. * **기능**: * `Internal Notes` 입력 필드 분리. * 슬롯 마감 시 시간 선택 드롭다운에서 해당 시간 딤드(Dimmed) 처리. * **예약금 관리 섹션** (9인 이상 노출): 상태/금액 수정 및 관련 메일 발송 기능. ### 5.4. 시스템 설정 (Tab: Settings) * **사용자 관리**: * 이메일 기반 사용자 추가/수정/삭제. * 지점별 접근 권한 매핑 (매니저 권한 시 필수). * **커스텀 슬롯 개편**: * 날짜 중심 **피벗 뷰 (Date-Pivot View)**. * 상단 날짜 네비게이션으로 날짜 이동. * 시간대별 슬롯 현황 조회 및 일괄 수정 모달 제공. * **메일 템플릿 관리 (Mail Templates) \[New\]**: * 대상: `mail_template` 시트에 저장된 데이터. * UI: 리스트에서 템플릿 선택 -> 편집 모달 오픈. * **편집기**: **Quill Editor** 적용 (HTML 모드/Preview 모드). * **저장 로직**: "저장" 버튼 클릭 시 HTML 문자열을 서버로 전송 -> `mail_template` 시트의 `body_html` 컬럼 업데이트. * **예약어 삽입**: `{{customer_name}}` 대신 `[[customer_name]]` 사용 (Vue 문법 충돌 방지). ## 6. 비즈니스 로직 (Business Logic) ### 6.1. 예약금 정책 (Deposit Policy) * **대상**: 9인 이상 예약. * **금액**: 9\~20명(\$100), 21\~30명($200), 31\~40명($300)... ($100 단위 증액). * **자동화**: `ReservationBatch` 실행 시 9인 이상이면 `deposit_status='pending'` 설정 및 안내 메일 발송. ### 6.2. 슬롯 확보 전략 * **로직**: 예약 확정/변경 시 `(Max Slot - Current Reservation)` 확인 후, 여유 있으면 위젯 캘린더 블로킹 해제, 마감이면 블로킹 생성. ### 6.3. 메일 발송 (Sheet Based Template) * **저장소**: `mail_template` 시트. * Key: `reservation_update`, `reservation_cancel`, `deposit_pending`, `deposit_confirm`, `deposit_refund` * **수정 흐름**: Web App (Quill) -> `WebApi.apiSaveSystemSettings('mail_template')` -> `MailTemplateService` -> 시트 갱신. * **발송 흐름**: `MailTemplateService.getTemplate(key)` -> 데이터 치환 -> `GmailApp.sendEmail`.
